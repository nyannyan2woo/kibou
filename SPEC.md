# マスメディア型防災アプリ 企画書

## 1. プロジェクト概要 (Overview)
本アプリは、災害時にSNS等で飛び交うデマやディープフェイク、インプレッション目的のノイズを完全に排除し、「迅速かつ信憑性の高い情報のみ」をユーザーに届けることに特化した「マスメディア型（一方向配信型）」の防災アプリケーションです。ユーザーからの直接発信機能をあえて削ぎ落とし、情報の受け取りと避難行動の支援に特化しています。

## 2. 解決する課題 (Problem)
* **情報のノイズとデマの拡散:** 大規模災害時におけるSNS上の偽情報やスパムによる、救助活動の妨げや避難行動の遅れ。
* **情報の分散:** 自治体、ライフライン、交通機関など、確認すべき公式情報が各所に散らばっており、パニック時にユーザーが探し出せない問題。
* **人的リソースの限界:** 人間によるファクトチェックや情報まとめサイトの更新では、災害の進行スピードに追いつけないタイムラグの発生。

## 3. コアバリュー (Core Values)
1. **ノイズレス (No Noise):** ユーザー発信を不可とし、情報の錯綜を防ぐ。
2. **AI完全自動化 (AI-Driven):** 24時間365日、AIが情報の収集・要約・配信を全自動で実行。
3. **透明性 (Transparency):** 情報の信憑性をシステム側で勝手に判断して捨てるのではなく、「信頼度レイヤー」として可視化し、ユーザーに判断材料を提供する。

## 4. 主要機能 (Key Features)

### A. 大規模災害時の「特設ページ」自動生成
* 地震（震度〇以上）や津波、特別警報などのトリガーを検知し、AIが自動で該当地域向けの「特設ページ」を構築。
* 避難所情報、給水所、交通機関の運行状況、ライフライン（停電・断水）の公式発表を１つの画面に集約し、情報のハブとして機能する。

### B. 信憑性レベル別UI（3層レイヤー構造）
集約した情報を、情報源の確実性に基づいて3つのTier（階層）に分類し、直感的なUIで表示する。

* **Tier 1: 確定情報【信頼度100%】**
  * **情報源:** 気象庁API、自治体公式発表、インフラ企業公式、特務機関NERV（L2接続データ）など。
  * **UI表現:** 緑や青など安心感・確実性を示すカラー。トップ画面に最優先で固定表示。
* **Tier 2: 報道・確認中情報【信頼度: 中〜高】**
  * **情報源:** 大手通信社、テレビ局の報道、公式メディアのSNS。
  * **UI表現:** 黄色など注意を促すカラー。「公式発表待ち・報道ベース」であることを明記。
* **Tier 3: 現場の速報・AI抽出【信頼度: 低】**
  * **情報源:** 一般SNSからAIが異常検知・抽出した「現場のトレンド（火災発生の噂など）」。
  * **UI表現:** グレーなど控えめなカラーに⚠️警告アイコンを付与。デマ拡散防止のため、**ワンタップしないと詳細が読めない**（認知のワンクッションを置く）設計。

### C. AIによる情報トリアージと要約
* ホワイトリスト化された情報源を24時間クローリング。
* 災害時のパニック状態でも読みやすいように、AIが長文の公式発表を「結論（どうすべきか）」から始まる短い文章に自動要約して配信。

## 5. ターゲットユーザー (Target Users)
* 災害時に「確実な情報だけ」をノイズなしで取得したい全スマートフォンユーザー。
* 情報過多によってパニックになりやすい高齢者や、デジタル情報の真偽判定（リテラシー）に不安がある層。

## 6. 今後の展望・ビジネスモデル (Future Prospects)
* **超パーソナライズ通知:** 位置情報を活用し、「ユーザーの現在地から最も近い開設済みの避難所」など、生存に直結する情報だけをプッシュ通知する機能の追加。
* **BtoB / BtoG 展開:** 本アプリの強力な「AI情報収集・精査エンジン」を、企業のBCP（事業継続計画）部門や、自治体の災害対策本部向けにダッシュボードとして提供し、マネタイズを図る。
